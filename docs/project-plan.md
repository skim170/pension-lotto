# 연금복권 분석 & 추천 프로젝트 기획서

기준일: 2026-03-22

## 1. 프로젝트 개요

`연금복권 분석 & 추천`은 최근 연금복권 720+ 당첨 결과를 바탕으로 당첨 흐름을 직관적으로 보여주고, 통계 기반 추천번호를 생성해 주는 정적 웹 서비스다. 사용자는 복잡한 회원가입이나 별도 설치 없이 웹페이지에서 최근 회차 결과를 확인하고, 조별 빈도와 자리수별 숫자 분포를 확인한 뒤 추천번호를 바로 복사할 수 있다.

이 프로젝트는 "정보 확인"과 "가벼운 재미"를 동시에 제공하는 것이 핵심이다. 따라서 서비스는 분석형 UI를 제공하되, 추천번호는 어디까지나 통계 기반 가중 랜덤이며 당첨 보장을 제공하지 않는다는 점을 명확히 유지한다.

## 2. 추진 배경

연금복권 사용자는 매주 당첨 결과를 확인하지만, 공식 사이트에서는 최근 흐름을 한 번에 분석해서 보기 어렵다. 특히 다음과 같은 불편이 존재한다.

- 최근 당첨 결과를 여러 회차 기준으로 비교하기 번거롭다.
- 조별/자리수별 패턴을 직관적으로 파악하기 어렵다.
- 단순 자동 추천보다 "최근 결과를 반영한 추천"에 대한 흥미 수요가 있다.

본 프로젝트는 이러한 사용 흐름을 단일 페이지 안에서 단순화해, 결과 확인부터 추천번호 생성까지의 시간을 줄이는 것을 목표로 한다.

## 3. 프로젝트 목표

### 핵심 목표

- 최근 30회 당첨 데이터를 한 화면에서 확인할 수 있게 한다.
- 조별 빈도와 자리수별 숫자 분포를 시각적으로 제공한다.
- 최근 데이터 분포를 반영한 추천번호 5개를 즉시 생성한다.
- 운영자가 수동으로 데이터를 갱신하지 않아도 되도록 자동 업데이트 구조를 만든다.

### 비목표

- 당첨 확률을 높여 준다고 보장하지 않는다.
- 머신러닝 예측 서비스로 포지셔닝하지 않는다.
- 결제, 회원제, 커뮤니티 기능은 현재 범위에 포함하지 않는다.

## 4. 타깃 사용자

### 1차 타깃

- 매주 연금복권 결과를 확인하는 일반 구매자
- 직접 번호를 고르기 전에 최근 흐름을 참고하고 싶은 사용자

### 2차 타깃

- 복권 데이터를 가볍게 분석해 보는 것을 즐기는 사용자
- 통계 기반 추천 콘텐츠를 재미 요소로 소비하는 사용자

## 5. 핵심 사용자 가치

- 최신 데이터 확인이 빠르다.
- 최근 30회 기준의 흐름을 시각적으로 이해하기 쉽다.
- 추천번호 생성과 복사가 즉시 가능하다.
- 별도 로그인, 설치, 복잡한 입력 없이 바로 사용할 수 있다.

## 6. 서비스 구성

현재 프로젝트는 단일 페이지 기반의 MVP로 구성되어 있으며, 아래 기능이 구현되어 있다.

### 1. 최근 30회 결과 조회

- `data/results.json`에서 당첨 결과를 읽어 온다.
- 최신 회차 기준으로 정렬한 뒤 최근 30회만 화면에 표시한다.
- 회차, 추첨일, 당첨번호를 테이블로 제공한다.

### 2. 조(1~5) 빈도 분석

- 최근 30회 결과를 기준으로 조별 출현 횟수를 계산한다.
- 막대 차트와 요약 문구를 함께 제공한다.
- 가장 많이 나온 조를 별도로 강조한다.

### 3. 자리수별 숫자 빈도 분석

- 각 회차의 6자리 번호를 자리별로 분해한다.
- 사용자가 1번째 자리부터 6번째 자리까지 선택해 분포를 볼 수 있다.
- 특정 자리에서 0~9 숫자가 얼마나 자주 등장했는지 차트로 보여 준다.

### 4. 추천번호 생성

- 최근 30회 데이터의 조별 빈도와 자리수별 숫자 빈도를 가중치로 사용한다.
- 가중 랜덤 방식으로 추천번호 5개를 생성한다.
- 추천번호 개별 복사 및 전체 복사를 지원한다.

### 5. 상태 피드백

- 데이터 로딩 상태 메시지 제공
- 토스트 메시지를 통한 성공/실패 피드백 제공

## 7. 사용자 이용 시나리오

1. 사용자가 웹페이지에 접속한다.
2. `최근 30회 불러오기` 버튼을 눌러 최신 데이터를 불러온다.
3. 조별 차트와 자리수별 차트를 확인하며 최근 흐름을 파악한다.
4. `추천번호 5개 생성` 버튼을 눌러 추천번호를 생성한다.
5. 원하는 추천번호를 개별 복사하거나 전체 복사해 구매에 활용한다.

## 8. 데이터 운영 방식

### 데이터 소스

- 동행복권 연금복권 결과 API 응답 데이터 사용

### 저장 포맷

- `round`: 회차
- `group`: 조 번호
- `num`: 6자리 당첨번호
- `date`: 추첨일

### 갱신 방식

- GitHub Actions를 통해 `data/results.json`을 주기적으로 업데이트한다.
- 스케줄 실행과 수동 실행(`workflow_dispatch`)을 모두 지원한다.
- 수집 스크립트는 수신 데이터를 정규화하고, 중복 회차 제거 후 최신 회차 순으로 저장한다.

### 운영 의의

- 프론트엔드는 정적 파일만 제공하면 되므로 운영 비용이 낮다.
- 데이터 갱신은 자동화되어 유지보수 부담이 작다.
- 백엔드 서버 없이도 최신 데이터 기반 서비스를 운영할 수 있다.

## 10. 현재 단계 평가

현재 프로젝트는 "작동 가능한 MVP" 단계로 볼 수 있다. 실제 당첨 데이터를 불러와 시각화하고 추천번호를 생성하는 핵심 가치가 이미 구현되어 있으며, 자동 데이터 갱신까지 구성되어 있어 개인 프로젝트 또는 배포형 마이크로 서비스로서 충분한 기본기를 갖추고 있다.

다만 아직은 다음 영역이 보완 대상이다.

- 서비스 소개, 사용 가이드, 주의 문구의 정보 구조 강화
- 초기 진입 시 자동 로딩 여부 및 로딩 경험 개선
- 모바일 사용성 세부 보완
- 추천 로직 설명력 강화
- 배포/접속 환경 기준 정리

## 11. 향후 개선 방향

### Phase 1. 운영 안정화

- 데이터 로딩 실패 시 재시도 안내 추가
- 마지막 갱신 시각 표기
- 추천 로직 설명 섹션 추가
- 빈 데이터 또는 API 구조 변경 시 예외 처리 강화

### Phase 2. 분석 고도화

- 최근 10회, 30회, 50회 기준 전환 기능
- 조별 장기 추세 비교
- 자리수별 핫/콜드 숫자 요약
- 회차 검색 및 특정 번호 필터

### Phase 3. 서비스 확장

- GitHub Pages 또는 정적 호스팅 기반 배포 체계 명확화
- 공유하기 기능
- 모바일 최적화 강화
- PWA 또는 알림형 기능 검토

## 12. 기대 효과

- 연금복권 결과 확인 경험을 더 빠르고 직관적으로 만든다.
- 단순 조회형 페이지보다 체류 시간이 긴 분석형 콘텐츠로 확장할 수 있다.
- 자동 업데이트 구조를 통해 적은 운영 비용으로 지속 가능한 서비스를 만들 수 있다.
- 향후 복권 데이터 기반 콘텐츠, 통계 실험, 개인 포트폴리오 프로젝트로도 확장 가능하다.

## 13. 결론

이 프로젝트는 "최근 연금복권 결과를 쉽게 분석하고, 이를 바탕으로 추천번호를 생성하는 경량 웹 서비스"라는 목적에 충실한 MVP다. 현재 구조만으로도 사용자에게 즉시 사용할 수 있는 가치를 제공하며, 특히 정적 프론트엔드와 자동 데이터 갱신의 조합 덕분에 운영 효율이 높다.

향후에는 분석 기준 다양화, 사용자 설명 강화, 배포 체계 정리를 중심으로 개선하면 단순 개인 실험을 넘어 공유 가능한 서비스 수준까지 자연스럽게 확장할 수 있다.
